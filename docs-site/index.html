Linux-first workflow for Vision Pro workspace development

Vision Web Workspace

Browser-simulated spatial web desktop with governed handoff to a real Mac Builder.

Vision Web Workspace 是一个面向 Apple Vision Pro 空间网页工作区的开发底座。它在 Linux 环境中先定义 workspace、window、session 和 Mac Builder contracts，用浏览器模拟可移动网页窗口、终端、代码编辑器和远程浏览器会话； 当需要原生 visionOS 构建、模拟器测试、归档或签名时，再把请求交给受控的远程 Mac + Xcode 执行面。

Simulate before Mac

先在浏览器中验证空间工作区交互。

Linux 开发环路覆盖窗口移动、缩放、聚焦、布局保存、终端和浏览器会话，不假装运行 Apple 官方模拟器。

Web windows as workspace

每个窗口都是可控网页工作面。

窗口可以承载 SSH terminal、code-server、文档、CI dashboard、日志或远程浏览器流，服务于真实开发流程。

Governed Mac handoff

原生构建交给明确的 Mac Builder 协议。

Linux 侧提交 build/test/archive 请求；macOS worker 负责 XcodeGen、xcodebuild、simulator、artifact 和 `.xcresult`。

Problem

没有 Mac 时，visionOS 开发不能从原生构建开始。

Apple 官方 visionOS 编译、签名、模拟器和设备测试依赖 macOS、Xcode、visionOS SDK 和开发者账号。 但产品设计、窗口模型、远程开发界面和构建请求协议不应该等到 Mac 环境全部就绪后才开始。

这个 repo 把可在 Linux 验证的部分前置：窗口状态、网页工作区、会话网关、workflow 检查和 Mac Builder 请求链路。 原生构建仍然由真实 Mac 承担，避免把浏览器模拟误写成 Apple 官方模拟器。

Product risk it reduces

在投入 EC2 Mac、MacBook、Xcode 签名和设备测试之前，先确认工作区交互、远程终端/浏览器体验和构建控制协议是否成立。

What it is

一个把空间网页工作区和 Mac 构建面分离的开发平台。

系统把产品交互、session gateway 和 native build handoff 拆成可验证的层。这样 Linux 可以推进大部分工作流， Mac 只承担 Apple 工具链必须承担的部分。

Workspace contracts

窗口和会话模型

Shared contracts 定义 workspace、panel、layout、session 和 Mac build request，不把状态散落在 UI 中。

Browser simulator

浏览器空间工作区

Web simulator 用普通浏览器验证 head-locked desktop、movable windows、terminal/code/browser/doc/log panels。

Session gateway

远程开发会话入口

Gateway skeleton 统一创建 terminal、code 和 browser session，后续可接 ttyd、code-server、noVNC 或 WebRTC。

Mac Builder handoff

原生构建控制链

Mock 和 agent packages 验证 build job、状态轮询、日志、artifact 和 `.xcresult` 返回格式。

What it is not

边界直接写在开发流程里。

这个 repo 不绕过 Apple 工具链，也不声称浏览器模拟等价于 Vision Pro 实机。

不是 Xcode、visionOS SDK、Apple 官方模拟器或签名系统的替代品。
不是 Enterprise API 或系统级 follow window entitlement 的替代方案。
不是生产远程运维平台；终端、浏览器和代码会话仍需要受控网关。
不是 App Store release authority；上传、签名和 TestFlight 仍属于后续 Mac/Apple workflow。
不把 SSH keys、GitHub tokens 或生产机器权限放进 Vision Pro client。
不声称 Linux 可以编译、安装或运行原生 visionOS app。

Operator workflow

本地开发和 Mac Builder 验证流程

本地流程先验证 contracts、simulator 和 gateway；native build check 可以连接 mock builder，也可以在真实 Mac Builder 存在时发起远程请求。

  1. 安装依赖：`pnpm install`。
  2. 启动 web simulator：`pnpm dev`。
  3. 运行 workflow 检查：`pnpm workflow:check`。
  4. 启动 mock builder：`pnpm dev:mac-builder:mock`。
  5. 提交 native build check：`VISIONOS_MAC_BUILDER_URL=http://127.0.0.1:3101 pnpm visionos:mac-build:check`。
