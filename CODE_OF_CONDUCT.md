# Code of Conduct

Vision Web Workspace is an Apple-platform development workflow project. Discussions should stay technical, evidence-based, and respectful.

## Expected behavior

- Keep feedback focused on code, workflow boundaries, tests, docs, and Apple tooling constraints.
- Be clear when a feature requires macOS, Xcode, signing assets, simulator access, or a real device.
- Avoid implying that Linux workflows can replace Apple-required native build and signing steps.
- Redact credentials, tokens, device identifiers, signing material, and private infrastructure details.

## Unacceptable behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing credentials, certificates, provisioning profiles, private keys, or production host details.
- Encouraging unsafe bypasses of Apple signing, approval, audit, or remote access boundaries.
- Repeated off-topic disruption of technical discussions.

## Enforcement

Maintainers may edit, hide, close, or lock issues, pull requests, discussions, or comments that violate this code of conduct.
